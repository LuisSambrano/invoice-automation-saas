# Contributing to Invoice Zero

## Minimalist Guidelines

1.  **Issues**: Please use English for technical issues. _Por favor usa inglés para temas técnicos si es posible._
2.  **Pull Requests**:
    - One feature per PR.
    - Follow the "Clean Architecture" pattern.
    - Include tests if applicable.

## Estándares (Español)

Buscamos mantener el código limpio y profesional ("Quiet Luxury").

- **Commits**: Conventional Commits (`feat:`, `fix:`, `docs:`).
- **Estilo**: Prettier y ESLint son obligatorios.
